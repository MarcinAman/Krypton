Apply the given x, y and z values to the given object by returning a new object with the applied values

This has the same signature as copy, but cannot be called copy due to Kotlin limitations

Generally, if you have an instance of either Vector or Location, you shouldn't need to use this, having this just allows us to provide implementations for most of the functions in this class
